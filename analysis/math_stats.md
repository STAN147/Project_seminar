## 1. Метрики расстояния между слоями

### 1.1 MSE (Mean Squared Error)

Среднеквадратичная ошибка между выходами слоёв i и j:

$$MSE(i, j) = \frac{1}{d} \sum_{k=1}^{d} (h_i^{(k)} - h_j^{(k)})^2$$

где d — размерность скрытого состояния, h — hidden state.

**Интерпретация:** чем больше MSE между соседними слоями, тем сильнее слой трансформирует представления → потенциально более важен.

### 1.2 Cosine Distance

Косинусное расстояние — мера угла между векторами:

$$CosDist(i, j) = 1 - \frac{\mathbf{h}_i \cdot \mathbf{h}_j}{\|\mathbf{h}_i\| \cdot \|\mathbf{h}_j\|}$$

- 0 — вектора сонаправлены (слои почти одинаковы)
- 1 — ортогональны
- 2 — противоположны

**Особенность:** нечувствительна к масштабу, измеряет только направление.

### 1.3 L1 Distance (Manhattan Distance)

$$L_1(i, j) = \frac{1}{d} \sum_{k=1}^{d} |h_i^{(k)} - h_j^{(k)}|$$

Более робастна к выбросам, чем MSE (нет возведения в квадрат).

### 1.4 L-Infinity Distance (Chebyshev Distance)

$$L_\infty(i, j) = \max_{k} |h_i^{(k)} - h_j^{(k)}|$$

Максимальное отклонение по одной координате. Чувствительна к выбросам — ловит экстремальные изменения в отдельных нейронах.

### 1.5 Residual Contribution

Относительный вклад слоя — насколько слой изменяет представление относительно его нормы:

$$ResContrib(i, j) = \frac{\|\mathbf{h}_j - \mathbf{h}_i\|}{\|\mathbf{h}_i\|}$$

- < 1 — изменение меньше самого сигнала
- \> 1 — слой радикально меняет представление

### 1.6 CKA (Centered Kernel Alignment)

Метрика сходства представлений, инвариантная к ортогональным преобразованиям и изотропному масштабированию:

$$CKA(X, Y) = \frac{\|Y^T X\|_F^2}{\|X^T X\|_F \cdot \|Y^T Y\|_F}$$

где X, Y — матрицы активаций (samples × features), F — норма Фробениуса.

- 1.0 — представления идентичны (с точностью до линейного преобразования)
- 0.0 — полностью различны

**Преимущество:** в отличие от MSE/Cosine, CKA сравнивает **структуру** представлений, а не конкретные значения.

### 1.7 Variance Ratio

Отношение дисперсий выходов:

$$VarRatio(i, j) = \frac{Var(\mathbf{h}_j)}{Var(\mathbf{h}_i)}$$

- 1.0 — масштаб представлений сохраняется
- \> 1 — слой увеличивает разброс
- < 1 — слой сжимает представления

### 1.8 Pearson Correlation

Линейная корреляция между поэлементными значениями выходов слоёв:

$$r(i, j) = \frac{\sum_{k}(h_i^{(k)} - \bar{h}_i)(h_j^{(k)} - \bar{h}_j)}{\sqrt{\sum_{k}(h_i^{(k)} - \bar{h}_i)^2 \cdot \sum_{k}(h_j^{(k)} - \bar{h}_j)^2}}$$

- 1.0 — полная положительная линейная зависимость
- 0.0 — нет линейной связи

### 1.9 Router Entropy

Энтропия распределения вероятностей роутера MoE-слоя:

$$H = -\sum_{e=1}^{E} p_e \log_2 p_e$$

где p_e — вероятность выбора эксперта e, E — число экспертов.

- Максимум (log2(E)) — равномерное использование экспертов
- Минимум (0) — всегда один эксперт

**В отличие от остальных:** не матрица 24×24, а вектор длины 24 (по одному значению на слой).

## 2. Прокси-метрика

Линейная комбинация нормализованных метрик, полученная градиентным спуском:

$$Proxy = \sum_{m} w_m \cdot \hat{x}_m + b$$

где нормализация — z-score:

$$\hat{x}_m = \frac{x_m - \mu_m}{\sigma_m}$$

Коэффициенты (из `proxy_metric.txt`):

| Метрика | Коэффициент |
|---------|-------------|
| L1_Dist | +0.7280 |
| Cosine_Dist | −0.6287 |
| Pearson | +0.6283 |
| L_Inf | +0.4512 |
| MSE | +0.2983 |
| Router_Entropy | +0.1063 |
| Res_Contrib | +0.0540 |
| Var_Ratio | +0.0400 |
| CKA | +0.0205 |

**Смысл:** Прокси-метрика аппроксимирует реальное падение точности на основе дешёвых в вычислении метрик (не нужно запускать eval на бенчмарке).

## 3. Корреляционный анализ

### 3.1 Корреляция Пирсона (Pearson r)

Измеряет **линейную** зависимость между двумя переменными:

$$r = \frac{\sum_{i}(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum_{i}(x_i - \bar{x})^2 \cdot \sum_{i}(y_i - \bar{y})^2}}$$

- Диапазон: [−1, +1]
- Предполагает: нормальность распределений, линейность связи
- Чувствительна к выбросам

### 3.2 Корреляция Спирмена (Spearman ρ)

Измеряет **монотонную** (не обязательно линейную) зависимость. Вычисляется как корреляция Пирсона на **рангах**:

1. Заменяем значения рангами: x → rank(x), y → rank(y)
2. Считаем Pearson r по рангам

- Диапазон: [−1, +1]
- Робастна к выбросам
- Не требует нормальности
- Ловит нелинейные монотонные зависимости (например, логарифмические)

### 3.3 Когда Pearson ≠ Spearman?

| Ситуация | Pearson | Spearman |
|----------|---------|----------|
| Линейная связь | Высокий | Высокий |
| Монотонная нелинейная (log, sqrt) | Умеренный | **Высокий** |
| Немонотонная (U-образная) | Низкий | Низкий |
| Выбросы | **Искажён** | Устойчив |

## 4. Статистическая значимость (p-value)

### 4.1 Что такое p-value?

p-value — вероятность наблюдать такую же или более сильную корреляцию **при условии, что истинная корреляция равна нулю** (H₀: ρ = 0).

- p = 0.03 → при отсутствии связи шанс увидеть такой r составляет 3%
- p < 0.05 → отвергаем H₀, связь статистически значима

### 4.2 Как вычисляется p-value для корреляции?

**Шаг 1.** Переводим r в t-статистику:

$$t = r \cdot \sqrt{\frac{n - 2}{1 - r^2}}$$

где n — число наблюдений (у нас n = 24 слоя), степени свободы df = n − 2 = 22.

**Шаг 2.** Вычисляем двусторонний p-value из t-распределения Стьюдента:

$$p = 2 \cdot P(T > |t| \mid df = n - 2)$$

Множитель 2 — потому что нас интересует и положительная, и отрицательная корреляция (двусторонний тест).

**Шаг 3.** Сравниваем с порогом:

| Обозначение | Порог | Интерпретация |
|-------------|-------|---------------|
| * | p < 0.05 | Значимо |
| ** | p < 0.01 | Высоко значимо |
| *** | p < 0.001 | Очень высоко значимо |
| (нет) | p >= 0.05 | Не значимо |

### 4.3 Пример

Для r = 0.6, n = 24:

```
t = 0.6 * sqrt(22 / (1 - 0.36))
  = 0.6 * sqrt(34.375)
  = 0.6 * 5.863
  = 3.518

p = 2 * P(T > 3.518 | df=22)
  = 2 * 0.00098
  = 0.00195   →   **
```

### 4.4 Критические значения r для n=24

| Порог | Минимальный |r| |
|-------|------------|
| p < 0.05 | 0.404 |
| p < 0.01 | 0.515 |
| p < 0.001 | 0.629 |

## 5. R² (коэффициент детерминации)

Используется в log-log графиках для оценки качества степенной зависимости.

$$R^2 = 1 - \frac{SS_{res}}{SS_{tot}} = 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}$$

- SS_res — сумма квадратов остатков (ошибки модели)
- SS_tot — полный разброс данных

Для простой линейной регрессии: **R² = r²** (квадрат корреляции Пирсона).

| R² | Качество модели |
|----|-----------------|
| > 0.9 | Отличное |
| 0.7 – 0.9 | Хорошее |
| 0.3 – 0.7 | Умеренное |
| < 0.3 | Слабое |

## 6. Log-log анализ

### Зачем?

Если зависимость имеет вид степенного закона:

$$y = a \cdot x^b$$

то в логарифмических координатах она становится **линейной**:

$$\log y = b \cdot \log x + \log a$$

- **slope (b)** — показатель степени. Показывает эластичность: при увеличении x в 10 раз, y изменяется в 10^b раз
- **intercept (log a)** — свободный член

### Что показывает в нашем контексте?

Если log-log график для метрики M и Accuracy Drop даёт хороший R²:
- Связь между M и важностью слоя описывается степенным законом
- slope > 0 → метрика растёт вместе с важностью
- slope < 0 → метрика обратно пропорциональна важности

## 7. Z-score нормализация

Используется для приведения метрик к единой шкале на итоговом графике и при вычислении прокси-метрики:

$$z_i = \frac{x_i - \mu}{\sigma}$$

где μ — среднее, σ — стандартное отклонение.

После нормализации: среднее = 0, стд. отклонение = 1. Это позволяет сравнивать метрики с разными масштабами (MSE ~ 0.01, L_Inf ~ 400) на одном графике.

## 8. Heatmap корреляционных матриц

Матрица корреляций NxN показывает попарную корреляцию между всеми переменными. Используется для:

1. **Выявления кластеров метрик** — группы метрик с высокой взаимной корреляцией измеряют одно и то же
2. **Поиска предикторов** — строка/столбец Abs_Drop показывает, какие метрики лучше предсказывают важность слоя
3. **Обнаружения мультиколлинеарности** — если две метрики коррелируют на 0.95+, одна из них избыточна в прокси-метрике

## 9. Ранговое сравнение (Rank Comparison)

### Зачем?

Для задачи pruning не нужно точно предсказывать падение точности — достаточно правильно определить **порядок** слоёв по важности. Ранговое сравнение оценивает именно это.

### Методика

**Шаг 1.** Каждому слою присваиваем два ранга:
- **Rank_Drop** — ранг по реальному падению точности (1 = наибольшее падение = самый важный)
- **Rank_Proxy** — ранг по значению прокси-метрики (1 = наибольшее значение)

**Шаг 2.** Вычисляем абсолютную разность рангов для каждого слоя:

$$RankDiff_i = |Rank_{Drop}(i) - Rank_{Proxy}(i)|$$

### Метрики качества ранжирования

#### Mean Rank Difference

$$\overline{RankDiff} = \frac{1}{N} \sum_{i=1}^{N} |Rank_{Drop}(i) - Rank_{Proxy}(i)|$$

- 0 — идеальное совпадение порядка
- Максимум зависит от N (для N=24 максимум ~12)

#### Top-K Overlap

Доля совпадения между K самыми важными слоями по реальному drop и по прокси-метрике:

$$TopK = \frac{|Top_K^{Drop} \cap Top_K^{Proxy}|}{K}$$

В нашем анализе используется K=5. Overlap 5/5 = идеальное попадание в важнейшие слои.

### Визуализация: Bump Chart

Bump chart (slopegraph) — соединяет линиями позицию каждого слоя в двух ранжировках:
- Параллельные линии → ранг сохраняется (хорошо)
- Пересекающиеся линии → ранги меняются (плохо)
- Цвет линии кодирует величину сдвига (зелёный → малый, красный → большой)

### Связь со Spearman

Корреляция Спирмена — это, по сути, агрегированная мера рангового соответствия (Pearson по рангам). Ранговое сравнение дополняет Spearman более детальной диагностикой: показывает **какие именно** слои ранжируются неправильно.
