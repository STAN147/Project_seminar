# Выводы из анализа прокси-метрик важности слоёв

## Контекст исследования

Исследуется возможность предсказания важности слоёв в MoE-моделях без запуска дорогого evaluation на бенчмарках. Вместо этого используется линейная комбинация дешёвых proxy-метрик (вычисляемых за один forward pass). Анализ проведён на двух моделях:

- **Qwen 1.5 MoE** — 24 слоя, 60 экспертов (прокси-метрика обучалась на этой модели)
- **Phi-Tiny** — 32 слоя (трансфер прокси-метрики без переобучения)

---

## 1. Качество прокси-метрики на Qwen (обучающая модель)

### Корреляция с реальным падением точности

| Метод | Значение | p-value |
|-------|----------|---------|
| Pearson r | 0.7112 | < 0.001 |
| Spearman rho | 0.7513 | < 0.001 |

Прокси-метрика объясняет ~50% дисперсии (R^2 ~ 0.50) падения точности при отключении слоя. Корреляция статистически значима.

### Ключевые наблюдения по отдельным метрикам (Qwen)

**Сильные предикторы (Spearman > 0.5):**
- Метрики расстояния (MSE, L1, L_Inf, Cosine Distance) — хорошо ловят первые ~6 слоёв, которые действительно критичны
- Pearson Correlation между соседними слоями
- Variance Ratio

**Слабые предикторы:**
- CKA — соседние слои почти всегда имеют CKA > 0.999, что делает метрику неинформативной на соседних парах
- Router Entropy — очень узкий диапазон значений у Qwen (4.086–4.094), нет дискриминативности
- Residual Contribution — низкий вес в прокси-метрике

### Блочная структура (из heatmap-анализа, Qwen)

Heatmap-ы выявили чёткие блоки слоёв:
- **Блок 0–5**: начальные слои, резко отличаются от остальных по всем метрикам
- **Блок 6–16**: средние слои, плавное изменение
- **Блок 17–22**: поздние слои, отдельный кластер
- **Слой 23**: выходной слой, аномально большие расстояния до всех остальных

Эта блочная структура согласуется с данными по accuracy drop: слои 0, 1, 3, 6 (начальные) и 12, 13 (переходные) — наиболее критичны.

## 2. Трансфер прокси-метрики на Phi-Tiny

### Корреляция Qwen-прокси-метрики с accuracy drop Phi-Tiny

| Метод | Значение | p-value | Значимость |
|-------|----------|---------|------------|
| Pearson r | 0.0511 | 0.7849 | Не значимо |
| Spearman rho | 0.6499 | 0.0001 | Значимо*** |

### Интерпретация расхождения Pearson vs Spearman

- **Pearson ~ 0** означает: нет линейной зависимости между прокси-метрикой и accuracy drop на Phi-Tiny
- **Spearman ~ 0.65** означает: ранговый порядок всё ещё хорошо сохраняется

Это указывает на **нелинейную монотонную** связь: прокси-метрика правильно ранжирует слои по важности, но абсолютные значения не пропорциональны.

### Причины деградации при трансфере

1. **Разная архитектура**: Qwen (24L) и Phi-Tiny (32L) имеют разную глубину и распределение важности
2. **Разное поведение Router Entropy**: у Qwen энтропия стабильно ~4.09 (почти равномерная маршрутизация), у Phi-Tiny — от 0.33 до 2.77 (сильная специализация экспертов)
3. **Разный профиль accuracy drop**: у Qwen несколько слоёв с катастрофическим drop (70%), у Phi-Tiny — более плавное распределение (максимум 65.5% только у слоя 0)

## 3. Сравнение профилей важности слоёв

### Qwen 1.5 MoE

- Резко выраженные критические слои: 0 (-70.7%), 1 (-70.1%), 6 (-66.9%), 12 (-61.4%), 3 (-58.0%)
- Много неважных слоёв с drop < 5%: слои 15–23 (кроме 17)
- Бимодальное распределение: слои либо очень важны, либо почти не важны

### Phi-Tiny

- Один критический слой: 0 (-65.5%)
- Выделяется слой 3 (-42.8%) и 14 (-13.1%)
- Основная масса слоёв имеет drop 1–8% — более равномерное распределение
- Поздние слои (24–31) почти безопасны для удаления (drop < 1%)

## 4. Выводы о применимости прокси-метрик

### Что работает

1. **Ранговое предсказание** — прокси-метрика на основе Qwen-коэффициентов корректно ранжирует слои по важности даже на другой архитектуре (Spearman = 0.65 на Phi-Tiny)
2. **Метрики расстояния** — L1 Distance, MSE и Cosine Distance — наиболее информативные отдельные метрики
3. **Линейная комбинация** даёт лучший результат, чем любая отдельная метрика
4. **Блочная структура** видна на heatmap-ах обеих моделей, что подтверждает возможность group pruning

### Что не работает

1. **Линейный трансфер** — абсолютные значения прокси-метрики не переносятся между архитектурами (Pearson ~ 0)
2. **Router Entropy** — неинформативна для Qwen (почти постоянна), но потенциально полезна для Phi-Tiny (большой разброс)
3. **CKA на соседних слоях** — слишком близка к 1.0, не различает слои

### Рекомендации

1. **Для pruning одной модели**: обучить прокси-метрику на самой модели (eval ~5 слоёв → получить ground truth → линейная регрессия → предсказать остальные)
2. **Для трансфера**: использовать только ранговый порядок, не абсолютные значения; рассмотреть переобучение весов на нескольких слоях целевой модели
3. **Для group pruning**: использовать heatmap-ы CKA и Cosine Distance для выявления блоков похожих слоёв, затем удалять целый блок
4. **Улучшение прокси-метрики**: рассмотреть нелинейные модели (полиномиальная регрессия, MLP), что может улучшить трансферабельность

---

## 5. Ограничения анализа

1. **Малая выборка**: n=24 (Qwen) и n=32 (Phi-Tiny) — ограниченная статистическая мощность
2. **Одна задача**: accuracy drop измерен только на common sense бенчмарке, на других задачах ранжировка может отличаться
3. **Single prompt dataset**: метрики вычислены на 1221 промптах — не гарантирована репрезентативность
4. **Только отключение слоёв**: реальный pruning может включать fine-tuning после удаления, что изменит порядок важности
5. **Линейность прокси-метрики**: предполагает аддитивность вкладов метрик, что может не выполняться
